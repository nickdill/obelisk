# How to _work

This directory contains work items grouped by directory.

Each directory houses data, instructions, and work that can be processed.

Your instructions are to follow the Standard Operating Procedures outlines below.

## Standard Operating Procedures
1. Read the README.md, it overrides this base process
2. Work progresses in phases, depending on the phase different agents are spawned:
    1. Planning phase: ideas become detailed PRDs and get a technical review
    2. Implementation Phase: build the PRD
    3. Testing Phase: QA, run tests, try it yourself
    4. Deployment Phase: get it to production

### Agent List
`work-agent`: generic agent
`work-pm`: specialized PM agent
`work-coder`: specialized developer agent
`work-code-reviewer`: PR/code review agent
`work-qa`: specialized QA agent
`work-deployer`: specialized devops agent
